# Django Internationalization (i18n) Patterns

**Translation workflows, locale handling, and multilingual Django applications**

## Django i18n Configuration

**CRITICAL**: Proper i18n setup enables multilingual applications and consistent user experience across languages.

### Basic i18n Setup

**✅ GOOD**: Configure Django for internationalization
```python
# settings.py
import os
from pathlib import Path

BASE_DIR = Path(__file__).resolve().parent.parent

# Internationalization
LANGUAGE_CODE = 'en'  # Default language
USE_I18N = True
USE_L10N = True
USE_TZ = True

LANGUAGES = [
    ('en', 'English'),
    ('es', 'Español'),
    ('fr', 'Français'),
    ('de', 'Deutsch'),
    ('lt', 'Lietuvių'),  # Lithuanian
]

LOCALE_PATHS = [
    BASE_DIR / 'locale',  # Project-wide translations
]

# Time zone settings
TIME_ZONE = 'UTC'  # Store in UTC
USE_TZ = True

# Date/time formatting
DATETIME_FORMAT = 'N j, Y, P'  # e.g., "Jan. 1, 2023, 12 a.m."
DATE_FORMAT = 'N j, Y'         # e.g., "Jan. 1, 2023"
TIME_FORMAT = 'P'              # e.g., "12 a.m."
```

## Translation Workflow

### Mark Strings for Translation

**✅ GOOD**: Use Django's translation functions
```python
from django.utils.translation import gettext as _, ngettext
from django.utils.translation import gettext_lazy as _lazy

# In views/models
def my_view(request):
    message = _("Welcome to our application!")
    return render(request, 'template.html', {'message': message})

# In models
class Article(models.Model):
    title = models.CharField(max_length=255, verbose_name=_lazy("Title"))
    content = models.TextField(verbose_name=_lazy("Content"))

# Pluralization
def show_results(count):
    message = ngettext(
        "Found %(count)d result",
        "Found %(count)d results",
        count
    ) % {'count': count}
    return message
```

**❌ BAD**: Hardcoded strings
```python
message = "Welcome to our application!"  # ❌ Not translatable
```

### Template Translation

**Use i18n template tags**:
```html
{# templates/base.html #}
{% load i18n %}

<h1>{% trans "Welcome" %}</h1>

<p>{% blocktrans with name=user.get_full_name %}
  Hello {{ name }}, welcome back!
{% endblocktrans %}</p>

{# Pluralization in templates #}
{% load i18n %}
{% blocktrans count count=articles|length %}
  There is {{ count }} article.
{% plural %}
  There are {{ count }} articles.
{% endblocktrans %}
```

**For client-side strings**:
```javascript

    // Django provides gettext in global scope for JS
```

## Translation Management

### Create Translation Files

**Generate and compile translations**:
```bash
# Create/update translation files
python manage.py makemessages -l es  # Spanish
python manage.py makemessages -l fr  # French
python manage.py makemessages -l de  # German
python manage.py makemessages -l lt  # Lithuanian

# Compile translations to .mo files
python manage.py compilemessages
```

### Rigid Translation Map Workflow (No Manual .po Editing)

For multi-locale projects, manually translating strings in `.po` files is tedious and prone to being overwritten or losing sync. Implement a **Translation Map Workflow** driven by scripts rather than manual file edits.

**Core Rule**: Developers should **never** manually edit `.po` files. Instead, curate translation dictionaries inside Python map files (e.g. `tools/translation_maps/*.py`), and run an automated compilation pipeline.

**Pipeline Steps** (usually wrapped in a Makefile):
1. **Extract** (`makemessages`): Pull new strings from code and templates.
2. **Fix Ownership/Format**: Standardize file permissions/formatting if needed.
3. **Fill**: A script (`fill_empty_po.py`) reads the curated Python translation maps and injects values into the `.po` files where `msgstr` is empty or fuzzy. Use a `FORCE_SYNC_MSGIDS` list for keys that need to be overwritten across the board.
4. **Audit**: An automated gate (`audit_translations.py`) that checks for invariants (e.g. trailing spaces, missing placeholders, fuzzy matches left over) and fails the pipeline if translations are invalid.
5. **Compile** (`compilemessages`): Generates the `.mo` files only if the audit passes.

*(Example implementation located in Teisutis at `docs/guides/translation_workflow.md`)*

### Translation File Format (.po)

**Example translation file**:
```po
# locale/es/LC_MESSAGES/django.po
msgid ""
msgstr ""
"Language: es\n"
"Content-Type: text/plain; charset=UTF-8\n"

msgid "Welcome"
msgstr "Bienvenido"

msgid "Hello %(name)s"
msgstr "Hola %(name)s"

msgid "Found %(count)d result"
msgid_plural "Found %(count)d results"
msgstr[0] "Encontrado %(count)d resultado"
msgstr[1] "Encontrados %(count)d resultados"
```

## Language Detection and Switching

### URL-based Language Detection

**Configure URL patterns for language switching**:
```python
# urls.py
from django.conf.urls.i18n import i18n_patterns
from django.urls import path, include

urlpatterns = [
    path('i18n/', include('django.conf.urls.i18n')),  # Language switching
]

# Internationalized URL patterns
urlpatterns += i18n_patterns(
    path('', include('blog.urls')),
    path('admin/', admin.site.urls),
    prefix_default_language=False,  # Don't prefix default language
)
```

**Language switching URLs**:
```
# English (default)
/
/articles/
/admin/

# Spanish
/es/
/es/articles/
/es/admin/

# French
/fr/
/fr/articles/
/fr/admin/
```

### Language Selector Template

**Create language switching widget**:
```html
{# templates/includes/language_selector.html #}
{% load i18n %}

<form action="{% url 'set_language' %}" method="post" id="language-form">
  {% csrf_token %}
  <input name="next" type="hidden" value="{{ redirect_to }}" />
  <select name="language" onchange="this.form.submit()">
    {% get_current_language as LANGUAGE_CODE %}
    {% get_available_languages as LANGUAGES %}
    {% for lang_code, lang_name in LANGUAGES %}
      <option value="{{ lang_code }}"{% if lang_code == LANGUAGE_CODE %} selected{% endif %}>
        {{ lang_name }}
      </option>
    {% endfor %}
  </select>
</form>
```

### Manual Language Switching

**Force language in views**:
```python
from django.utils import translation
from django.utils.translation import activate

def switch_language(request, language_code):
    """Manually switch language."""
    if language_code in [code for code, name in settings.LANGUAGES]:
        activate(language_code)
        request.session['django_language'] = language_code
    return redirect(request.META.get('HTTP_REFERER', '/'))

def temporary_translation(request):
    """Temporarily switch language for a block."""
    with translation.override('es'):
        message = _("Hello World")  # Will be in Spanish
        # ... do something
```

## Locale-Specific Formatting

### Date/Time Formatting

**Handle time zones and locale formatting**:
```python
from django.utils import timezone
from django.utils.formats import localize
from django.utils.translation import get_language

def format_datetime(dt):
    """Format datetime with user's locale."""
    if timezone.is_naive(dt):
        dt = timezone.make_aware(dt)
    
    # Convert to user's timezone (if stored)
    user_tz = request.user.timezone if request.user.is_authenticated else None
    if user_tz:
        dt = dt.astimezone(timezone.pytz.timezone(user_tz))
    
    # Format with locale
    return localize(dt, use_l10n=True)

# Usage in templates
{{ article.created_at|localize }}
```

### Number Formatting

**Locale-aware number formatting**:
```python
from django.utils.formats import localize, localize_input

def format_price(price):
    """Format price with locale-specific separators."""
    return localize(price, use_l10n=True)

def parse_price(input_string):
    """Parse user input with locale-specific format."""
    return localize_input(input_string, use_l10n=True)

# Template usage
{{ product.price|localize }}
```

## Time Zone Handling

### User-Specific Time Zones

**Store and use user time zones**:
```python
# models.py
class UserProfile(models.Model):
    user = models.OneToOneField(User, on_delete=models.CASCADE)
    timezone = models.CharField(
        max_length=50,
        default='UTC',
        choices=[(tz, tz) for tz in pytz.common_timezones]
    )

# middleware.py
class TimezoneMiddleware:
    """Set user's timezone from profile."""
    
    def __init__(self, get_response):
        self.get_response = get_response
    
    def __call__(self, request):
        if request.user.is_authenticated:
            try:
                profile = request.user.userprofile
                timezone.activate(profile.timezone)
            except UserProfile.DoesNotExist:
                timezone.deactivate()  # Fall back to settings.TIME_ZONE
        else:
            timezone.deactivate()
        
        response = self.get_response(request)
        return response
```

## Translation Testing

### Test Translated Strings

**Verify translations work correctly**:
```python
# tests/test_i18n.py
from django.test import TestCase, override_settings
from django.utils.translation import gettext as _, activate
from django.utils import translation

class TranslationTest(TestCase):
    
    @override_settings(LANGUAGE_CODE='en')
    def test_english_translations(self):
        """Test English translations."""
        activate('en')
        self.assertEqual(_("Welcome"), "Welcome")
    
    @override_settings(LANGUAGE_CODE='es')
    def test_spanish_translations(self):
        """Test Spanish translations."""
        activate('es')
        self.assertEqual(_("Welcome"), "Bienvenido")
    
    def test_pluralization(self):
        """Test pluralization works."""
        with translation.override('en'):
            self.assertEqual(
                ngettext("%(count)d item", "%(count)d items", 1) % {'count': 1},
                "1 item"
            )
            self.assertEqual(
                ngettext("%(count)d item", "%(count)d items", 2) % {'count': 2},
                "2 items"
            )
```

### Test Locale Formatting

**Verify date/number formatting**:
```python
from django.test import TestCase
from django.utils.formats import localize, localize_input
from django.utils import translation

class LocaleFormattingTest(TestCase):
    
    def test_date_formatting(self):
        """Test date formatting in different locales."""
        dt = datetime(2023, 12, 25, 15, 30)
        
        with translation.override('en'):
            formatted = localize(dt.date())
            self.assertIn('Dec', formatted)  # English format
        
        with translation.override('de'):
            formatted = localize(dt.date())
            self.assertIn('Dez', formatted)  # German format

            self.assertEqual(formatted, '1,234.56')
        
        with translation.override('de'):
            formatted = localize(1234.56)
            self.assertEqual(formatted, '1.234,56')
```

## Common i18n Patterns

### Model Translation

**For translatable model fields**:
```python
# Use django-modeltranslation
# https://django-modeltranslation.readthedocs.io/

# models.py
from modeltranslation import register
from modeltranslation.admin import TranslationAdmin

class Article(models.Model):
    title = models.CharField(max_length=255)
    content = models.TextField()

# Register for translation
register(Article, fields=['title', 'content'])

# admin.py
class ArticleAdmin(TranslationAdmin):
    pass

admin.site.register(Article, ArticleAdmin)
```

### Form Translation

**Translate form labels and messages**:
```python
# forms.py
from django import forms
from django.utils.translation import gettext_lazy as _

class ContactForm(forms.Form):
    name = forms.CharField(label=_("Your Name"))
    email = forms.EmailField(label=_("Email Address"))
    message = forms.CharField(widget=forms.Textarea, label=_("Message"))
    
    def clean_name(self):
        name = self.cleaned_data['name']
        if len(name) < 2:
            raise forms.ValidationError(_("Name must be at least 2 characters long"))
        return name
```

### Email Translation

**Translate email content**:
```python
# emails.py
from django.core.mail import send_mail
from django.template.loader import render_to_string
from django.utils.translation import gettext as _

def send_welcome_email(user):
    """Send localized welcome email."""
    subject = _("Welcome to our platform!")
    context = {
        'user': user,
        'site_name': _("Our Platform"),
    }
    
    # Render template with user's language
    with translation.override(user.profile.language):
        html_message = render_to_string('emails/welcome.html', context)
        plain_message = render_to_string('emails/welcome.txt', context)
    
    send_mail(
        subject,
        plain_message,
        'noreply@example.com',
        [user.email],
        html_message=html_message,
    )
```

## i18n Best Practices

### Development Workflow
- **Always use _lazy for model/field names**: Prevents translation at import time
- **Mark all user-facing strings**: Even error messages and buttons
- **Test in multiple languages**: Ensure layouts work with longer text
- **Use meaningful msgids**: Make translation context clear

### Translation Management
- **Regular translation updates**: Keep translations current
- **Professional translation**: Use services for quality translations
- **Translation memory**: Reuse common translations
- **Context comments**: Add comments for translators

### Performance Considerations
- **Lazy translation**: Use _lazy for strings translated at runtime
- **Translation caching**: Cache compiled translations
- **Minimal string concatenation**: Avoid building translatable strings dynamically

### Security Considerations
- **Escape translated content**: Never trust user input in translations
- **Validate language codes**: Prevent invalid language injection
- **Secure translation files**: Don't allow user uploads of .po files
